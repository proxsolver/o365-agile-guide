Case 04 | R&M Cost

💰 외주 정비 비용 투명화

외주 업체별 작업 이력과 비용을 체계적으로 관리하여 최적의 파트너사를 선정합니다

업체 평가
비용 절감
품질 향상

문제 상황

현장의 목소리

구매팀 최OO 대리가 외주비 정산 중 문제를 발견합니다:
"외주 정비 업체가 5곳인데, 어디가 잘하고 어디가 비싼지 비교가 안 됩니다. A업체는 싸게 하는데 재작업이 많고, B업체는 비싸지만 품질이 좋아요. 근데 이게 감으로 느끼는 거지, 숫자로 증명할 수가 없어요. 작업 이력이 엑셀, 종이 견적서, 이메일에 흩어져 있어서 업체 평가가 불가능합니다."

연간 외주비 2.4억원
목표 절감 20%
개선 기간 4주

📋 실행 가이드

1
SharePoint Lists
Week 1 (월~화)

외주 작업 관리 Lists 상세 구축

실행 순서

1-1
Lists 생성 위치 및 방법
📍 경로:
Teams → TS팀 → 채널: 정비관리 → 상단 탭 "+" 클릭 → Lists 선택

⚙️ 설정:
• "새 목록" 선택 → 이름: "외주작업 이력"
• "Teams 채널에서 표시" 체크
• 색상: 주황색 (R&M 테마)
1-2
컬럼 상세 설정 (총 12개)

각 컬럼의 유형, 필수 여부, 선택값, 유효성 검사까지 정확히 설정합니다.

컬럼 구성 상세:
컬럼명 유형 필수 상세 설정
작업번호 한 줄 텍스트 ✓ 형식: OUT-2025-0001 (자동채번은 Power Automate에서)
업체명 선택 ✓ 선택값: A산업, B테크, C정밀, D엔지니어링, E서비스
작업일 날짜 ✓ 기본값: 오늘 날짜
설비코드 조회 (Lookup) ✓ 원본: 설비마스터 Lists의 "설비코드" 컬럼
작업구분 선택 ✓ 선택값: 수리, 점검, 설치, 교체, 기타
작업내용 여러 줄 텍스트 ✓ 서식 있는 텍스트 사용
견적금액 통화 ✓ 소수 자릿수: 0, 통화: ₩
실제금액 통화 작업 완료 후 입력
차이(계산) 계산됨 - 수식: =[실제금액]-[견적금액]
작업시간(hr) 숫자 소수점 1자리 허용
품질평가 선택 선택값: ⭐⭐⭐⭐⭐(5점), ⭐⭐⭐⭐(4점), ⭐⭐⭐(3점), ⭐⭐(2점), ⭐(1점)
재작업여부 예/아니요 기본값: 아니요
1-3
"조회(Lookup)" 컬럼 만들기 상세
💡 Lookup 컬럼이란?
다른 Lists의 데이터를 참조하여 선택할 수 있게 하는 컬럼입니다. 수동 입력 오류를 방지하고 데이터 일관성을 유지합니다.

설정 방법:
1. Lists 설정 → 열 만들기 → "조회" 선택
2. 열 이름: 설비코드
3. 정보 원본: 설비마스터 (같은 사이트의 다른 Lists)
4. 이 열: 설비코드
5. ✓ "다음 필드도 추가": 설비명 체크 (자동으로 같이 표시됨)
1-4
"계산됨(Calculated)" 컬럼 수식
// 차이 금액 계산
=[실제금액]-[견적금액]

// 차이율 계산 (별도 컬럼 추가 시)
=IF([견적금액]=0,0,([실제금액]-[견적금액])/[견적금액]*100)

// 시간당 비용 계산
=IF([작업시간]=0,0,[실제금액]/[작업시간])
1-5
뷰(View) 설정으로 데이터 쉽게 보기
생성할 뷰 3가지:
뷰 이름 필터 조건 그룹화 용도
월별 현황 작업일 = 이번 달 업체명 월간 업체별 실적 확인
재작업 목록 재작업여부 = 예 업체명 품질 문제 업체 파악
고비용 작업 실제금액 > 500만원 없음 대형 작업 모니터링
📍 뷰 만드는 법:
Lists 우측 상단 "모든 항목 ▼" 클릭 → "새 보기 만들기" → 조건 설정 후 저장
💡 Tip: Lists 설정 → 유효성 검사에서 =IF([실제금액]>[견적금액]*1.2, FALSE, TRUE)를 추가하면 견적 대비 120% 초과 시 경고가 표시됩니다.

Power Automate
Week 1 (수~금)

자동 번호 생성 및 알림 흐름 구축

흐름 1: 작업번호 자동 생성

2-1
새 흐름 만들기
📍 경로:
Power Automate (flow.microsoft.com) → 만들기 → 자동화된 클라우드 흐름

⚙️ 기본 설정:
• 흐름 이름: "외주작업_번호자동생성"
• 트리거 선택: "항목이 만들어질 때" (SharePoint)
2-2
흐름 단계별 상세 구성
1
트리거: 항목이 만들어질 때
커넥터: SharePoint
사이트 주소: https://회사.sharepoint.com/sites/TS팀
목록 이름: 외주작업 이력
↓
2
작업: 항목 가져오기 (최근 번호 조회)
커넥터: SharePoint - 여러 항목 가져오기
필터 쿼리: startswith(작업번호,'OUT-2025')
정렬: 작업번호 desc
상위 개수: 1
↓
3
작업: 변수 초기화 (새 번호)
이름: NewNumber
형식: 문자열
값:
concat('OUT-', formatDateTime(utcNow(), 'yyyy'), '-',
  formatNumber(add(int(substring(
    first(body('여러_항목_가져오기')?['value'])?['작업번호'],
    9, 4)), 1), '0000'))
→ 결과: OUT-2025-0001, OUT-2025-0002, ...
↓
4
작업: 항목 업데이트
커넥터: SharePoint - 항목 업데이트
ID: @{triggerOutputs()?['body/ID']}
작업번호: @{variables('NewNumber')}

⏰ 트리거: 항목을 만들거나 수정할 때
↓
🔍 조건: [견적금액] >= 5000000 (500만원 이상)
예 ↓

Excel
Week 2 (월~수)

업체별 성과 분석 대시보드

3-1
Lists 데이터 Excel로 내보내기
📍 방법 1: 일회성 내보내기
Lists 상단 "내보내기" → "Excel로 내보내기"

📍 방법 2: 실시간 연결 (권장)
Excel 열기 → 데이터 탭 → "데이터 가져오기" → "SharePoint 목록에서" → 사이트 URL 입력 → 목록 선택 → "로드"

※ 실시간 연결 시 데이터 새로 고침으로 최신 데이터 반영
3-2
업체별 성과 지표 피벗 테이블
피벗 테이블 설정:
영역 필드 설정
행 업체명 -
값 작업번호 개수 (작업 건수)
값 실제금액 합계 (총 비용)
값 품질평가 평균 (평균 점수)
값 재작업여부 개수 (재작업 건수)
3-3
업체 종합 평가 점수 산출
// 종합평가 점수 (100점 만점)

= (품질점수×0.4) + (비용효율×0.3) + (재작업감점) + (납기점수×0.2)

// 각 항목 수식
품질점수 = 평균품질평가 × 20 // 5점→100점
비용효율 = (1 - 평균차이율) × 100 // 견적 대비 절감할수록 높음
재작업감점 = 재작업건수 / 총건수 × -10
납기점수 = 100 // 지연 시 -10점씩

🏆 A산업 (87점)

품질 우수, 비용 적정, 재작업 1건

→ 핵심 파트너 유지

⚠️ C정밀 (62점)

비용 저렴, 품질 보통, 재작업 5건

→ 개선 요청 또는 물량 축소
