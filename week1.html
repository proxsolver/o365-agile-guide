🏗️ 환경 구축

Agile 실천을 위한 Office 365 기반 협업 환경을 구축합니다

이번 주 목표

Agile 팀이 협업할 수 있는 디지털 워크스페이스를 구축합니다. 모든 팀원이 동일한 공간에서 소통하고, 작업을 관리하며, 문서를 공유할 수 있는 환경을 만듭니다.

예상 소요 시간: 2~4시간 (초기 설정)

📋 단계별 실행 가이드

1
Microsoft Teams
30분

Teams 팀 및 채널 생성

왜 필요한가요?

Teams는 Agile 팀의 커뮤니케이션 허브입니다. Daily Standup, Sprint Planning 등 모든 스크럼 이벤트가 여기서 진행되며, 팀원 간 실시간 소통이 가능합니다.

실행 순서

1-1
Teams 앱 실행

Windows: 시작 메뉴에서 "Microsoft Teams" 검색 후 실행
웹: teams.microsoft.com 접속

1-2
새 팀 만들기

왼쪽 메뉴에서 "팀" 클릭 → 하단의 "팀 참가 또는 만들기" 클릭 → "팀 만들기" 선택

💡 Tip: "처음부터 만들기"를 선택하고, 팀 유형은 "비공개"를 권장합니다.
1-3
팀 이름 및 설명 입력

예시:
• 팀 이름: [프로젝트명] Agile Team
• 설명: 스크럼 기반 Agile 개발팀 협업 공간

1-4
팀원 초대

팀 생성 후 나타나는 화면에서 팀원 이메일 입력하여 초대
(나중에 추가 가능: 팀 이름 옆 "..." → "구성원 추가")

1-5
채널 구성

기본 "일반" 채널 외에 다음 채널들을 추가로 생성하세요:

  • 📋 Sprint Planning - 스프린트 계획 논의
  • 🔄 Daily Standup - 일일 스탠드업 미팅
  • 🎯 Sprint Review - 스프린트 리뷰 및 데모
  • 💡 Retrospective - 회고 미팅
  • 📚 Resources - 참고 자료 공유

채널 추가 방법: 팀 이름 옆 "..." → "채널 추가"

완료 체크리스트

2
Microsoft Planner
20분

Planner 칸반 보드 설정

왜 필요한가요?

Planner는 시각적 태스크 관리 도구입니다. Jira나 Trello와 유사하게 칸반 보드 형태로 작업 진행 상황을 한눈에 파악할 수 있습니다.

실행 순서

2-1
Teams에서 Planner 탭 추가

Teams의 "일반" 채널 상단 "+" 버튼 클릭 → "Planner" 검색 후 선택 → "새 계획 만들기" 선택

💡 Tip: Teams 내에서 Planner를 추가하면 자동으로 팀과 연동됩니다.
2-2
계획(플랜) 이름 설정

예시: Sprint Board - [프로젝트명]

2-3
버킷(열) 구성

기본 "To do" 버킷을 클릭하여 다음과 같이 버킷을 구성하세요:

📋 Backlog

스프린트에 들어오기 전

📌 To Do

이번 스프린트 할 일

🔄 In Progress

진행 중인 작업

👀 Review

검토 대기 중

✅ Done

완료된 작업

새 버킷 추가: 오른쪽 "새 버킷 추가" 클릭

2-4
레이블 설정

상단 "..." → "플랜 설정" → 레이블 탭에서 색상별 의미 지정

  • 긴급 - 즉시 처리 필요
  • 중요 - 높은 우선순위
  • 기능 - 새 기능 개발
  • 버그 - 버그 수정
  • 개선 - 기존 기능 개선

완료 체크리스트

3
SharePoint
30분

SharePoint 문서 라이브러리 구조화

왜 필요한가요?

SharePoint는 팀의 중앙 문서 저장소입니다. 프로젝트 산출물, 스프린트 결과물, 회의록 등 모든 문서를 버전 관리하며 저장합니다.

실행 순서

3-1
SharePoint 사이트 접근

Teams 팀을 생성하면 SharePoint 사이트가 자동 생성됩니다.
Teams 상단 채널 이름 옆 "파일" 탭 → "SharePoint에서 열기" 클릭

3-2
폴더 구조 생성

"+ 새로 만들기" → "폴더"를 선택하여 다음 구조를 만드세요:

📁 01_Product_Backlog
📁 02_Sprint_Deliverables
Sprint_01
Sprint_02
(스프린트별 폴더 추가)
📁 03_Meeting_Notes
📁 04_Templates
📁 05_References
3-3
버전 기록 확인

파일 우클릭 → "버전 기록"에서 이전 버전 확인 가능
(기본적으로 500개 버전까지 자동 저장됨)

💡 Tip: 중요한 문서는 파일명에 날짜를 붙이지 않아도 됩니다. SharePoint가 자동으로 버전을 관리해줍니다.
3-4
Teams에서 바로가기 추가

자주 사용하는 폴더를 Teams 탭에 추가:
채널 상단 "+" → "SharePoint" 또는 "문서 라이브러리" 선택

완료 체크리스트

4
OneNote
20분

OneNote 팀 전자 필기장 생성

왜 필요한가요?

OneNote는 팀의 회의록 및 아이디어 허브입니다. 비정형 정보, 브레인스토밍, 회고 내용 등을 자유롭게 기록하고 공유할 수 있습니다.

실행 순서

4-1
Teams에서 OneNote 탭 추가

채널 상단 "+" 버튼 → "OneNote" 검색 후 선택 → "새 전자 필기장 만들기"

4-2
전자 필기장 이름 설정

예시: Agile Team Notebook - [프로젝트명]

4-3
섹션 구성

다음 섹션(탭)들을 생성하세요:

📋 Sprint Planning
🔄 Daily Standup
🎯 Sprint Review
💡 Retrospective
📝 Meeting Notes
💭 Ideas & Brainstorm

섹션 추가: 섹션 탭 영역에서 우클릭 → "새 섹션"

4-4
템플릿 페이지 생성

각 섹션에 템플릿 페이지를 미리 만들어두면 편리합니다:

Daily Standup 템플릿 예시:
📅 날짜: [YYYY-MM-DD]
👥 참석자: 

---
🟢 어제 한 일:
• 

🔵 오늘 할 일:
• 

🔴 장애물/블로커:
• 

---
📝 기타 논의:
